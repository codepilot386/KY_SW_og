# 1단계 : 사용자 입력 및 인터페이스
## 흐름 : 어르신 및 정보 취약계층 사용자가 프론트엔드(HTML/CSS/JS)를 통해 음성이나 버튼 클릭으로 응답을 보냅니다.
## 출력 방식 : VS Code로 구축된 단정한 UI와 큰 글씨를 통해 사용자의 다음 행동을 유도합니다.

# 2단계 : 엔진 레이어의 동시 처리
## 프론트엔드에서 넘어온 데이터는 세 가지 엔진에서 목적에 맞게 가공됩니다.
## 시나리오 엔진 : 사용자의 상황에 맞는 대본 데이터를 추출하거나 생성합니다.
## 압박 엔진 : 실시간으로 타임 어택 데이터를 생성하여 화면을 붉게 만들거나 경고음을 출력해 긴장감을 조성합니다.
## 피드백 엔진 : 사용자의 답변이 오답인지 실시간으로 판별하는 데이터를 준비합니다.

# 3단계 : AI레이어 분석
## 흐름 : 가공된 데이터가 Geminu API로 전달됩니다.
## 역할 : AI는 사용자의 응답 패턴을 분석하고, 시나리오를 동적으로 생성하며, 최종 취약점 리포트를 위한 지표를 산출합니다.

# 4단계 : 데이터베이스 저장
## 분석된 결과물은 성격에 따라 세 가지 DB에 저장됩니다.
## 시나리오 DB : 대본 및 스크립트 데이터 업데이트.
## 사용자 이력 DB : 사용자의 훈련 기록 및 반복된 취약점 이력 저장.
## 분석 결과 DB : 개인별 피드백 및 상세 보고서 데이터 저장.

# 5단계 : 최종 결과 출력
## 흐름 : DB에 저장된 데이터를 바탕으로 프론트엔드 화면에 최종 결과가 출력됩니다.

## 즉시 출력 : 방금 선택은 위험했습니다. 와 같은 즉시 오답 피드백.
## 최종 출력 : 훈련 종료  후 제공되는 개인별 취약점 정밀 분석 보고서.
